# Sub-Agent 전략

> 작성일: 2026-04-18

---

## Sub-Agent란?

Claude Code에서 메인 Claude가 **별도 컨텍스트를 가진 Claude 인스턴스**를 생성해 병렬로 작업을 위임하는 기능.
메인 Claude의 컨텍스트 창을 보호하고, 독립적인 작업을 동시에 진행해 속도를 높이는 데 유리하다.

---

## Sub-Agent가 유리한 경우

| 시나리오 | 이유 |
|---------|------|
| 대규모 코드베이스 탐색 | 탐색 결과가 메인 컨텍스트를 오염시키지 않음 |
| 독립적인 파일 다수 생성 | 병렬 처리로 속도 향상 |
| 리서치 + 구현 동시 진행 | 각 에이전트가 전문화된 컨텍스트 유지 |
| 테스트 실행 & 분석 | 테스트 출력이 길어도 메인 창 보호 |

## Sub-Agent가 불필요한 경우

| 시나리오 | 이유 |
|---------|------|
| 순서 의존적인 작업 | A가 끝나야 B를 시작할 수 있으면 병렬 무의미 |
| 소규모 파일 수정 | 오버헤드가 더 큼 |
| 단순 Q&A / 설명 | 메인 Claude로 충분 |

---

## 이 프로젝트에서의 Sub-Agent 활용 평가

### 결론: **제한적으로 적합**

이유:
- 이 프로젝트는 학습 목적의 **소규모 샘플**이다
- 파일 수가 적고, 작업 흐름이 대부분 **순서 의존적** (빌드 → 로드 → 테스트)
- 과도한 에이전트 분할은 오히려 컨텍스트 공유가 안 돼 혼선 유발

### 실제로 유용한 Sub-Agent 시나리오 (이 프로젝트 기준)

```
시나리오 1: 초기 세팅
  메인 에이전트: Blazor 프로젝트 생성 + manifest.json 작성
  서브 에이전트A: NuGet 패키지 조사 (Blazor.BrowserExtension 버전 확인)
  → 병렬 가능 ✅

시나리오 2: 기능 추가 시
  메인 에이전트: 구현 진행
  서브 에이전트A: Chrome Extension API 공식 문서 리서치
  → 병렬 가능 ✅

시나리오 3: 빌드 + 테스트
  메인 에이전트: 빌드 실행
  서브 에이전트A: dist/ 파일 구조 검증
  → 작업이 워낙 짧아 직렬로도 충분 ⚠️
```

---

## 제안하는 에이전트 역할 정의

| 에이전트 | 역할 | 타입 |
|---------|------|------|
| **메인** | 구현, 파일 편집, 빌드 실행 | 항상 활성 |
| **Researcher** | NuGet/npm 패키지 조사, Chrome API 문서 탐색 | 필요 시 `Explore` 서브에이전트 |
| **Validator** | dist/ 구조 검증, manifest.json 스키마 확인 | 필요 시 `general-purpose` 서브에이전트 |

---

## 사용 방법 예시

메인 Claude가 아래와 같이 서브에이전트를 띄울 수 있다:

```
Agent(subagent_type="Explore"):
  "Find the latest stable version of Blazor.BrowserExtension NuGet package
   and check if it supports .NET 10"
```

복잡도가 올라가는 시점(예: Content Script 연동, Storage API 구현)에서 적극 활용 예정.
