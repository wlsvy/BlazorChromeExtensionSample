# Chrome Web Store 배포 조건

> 질문: 배포를 위해 크롬 확장 마켓 플레이스? 제출 조건이 따로 있는지?
> 작성일: 2026-04-18

---

## Chrome Web Store란?

Google이 운영하는 Chrome Extension/App 공식 배포 플랫폼.
`chrome://extensions/`에서 로컬 로드는 개발자만 가능하지만,
일반 사용자에게 배포하려면 Web Store를 거쳐야 한다.

---

## 등록 절차 요약

```
1. 개발자 계정 등록 (일회성 $5 수수료)
2. .zip 파일 패키징
3. Web Store 대시보드에 업로드
4. 심사 (자동 + 수동) → 보통 수일 내 완료
5. 게시
```

---

## 제출 전 필수 조건

### 1. 개발자 계정
- Google 계정 필요
- **등록비: $5 (USD, 일회성)**
- [Chrome Web Store Developer Dashboard](https://chrome.google.com/webstore/devconsole/)

### 2. 기술 요건

| 항목 | 요건 |
|------|------|
| Manifest 버전 | **V3 필수** (V2는 신규 등록 불가, 2024년부터) |
| 최소 권한 원칙 | 실제로 사용하는 권한만 `permissions`에 명시 |
| 원격 코드 실행 금지 | 외부 URL에서 스크립트 fetch 후 실행 불가 |
| CSP 준수 | 인라인 스크립트 금지 |
| HTTPS | 외부 리소스는 HTTPS만 허용 |

### 3. 스토어 등록 자료

| 항목 | 규격 |
|------|------|
| 아이콘 | 128×128px PNG (필수), 48×48, 16×16도 권장 |
| 스크린샷 | 최소 1장, 1280×800 또는 640×400 |
| 설명 | 영문 포함 권장 (다국어 지원 가능) |
| 카테고리 선택 | Productivity, Developer Tools 등 |
| 개인정보처리방침 | 데이터 수집 시 URL 필수 (수집 없으면 불필요) |

### 4. 심사 정책 주요 항목
- 악성 코드, 스파이웨어, 광고 삽입 금지
- 사용자 동의 없이 데이터 수집·전송 금지
- 제목/설명에 오해를 유발하는 키워드 남용 금지
- 다른 확장의 기능을 복제하거나 사칭 금지

## 패키징 방법

```bash
# dist/ 폴더 내용을 zip으로 묶기
# 주의: dist/ 폴더 자체가 아니라 내부 파일들이 zip 루트에 위치해야 함
cd dist/
zip -r ../extension.zip .
```

업로드 시 `.zip` 파일 하나를 대시보드에 드래그 앤 드롭하면 된다.

---

## 공부용 프로젝트라면?

Web Store 등록 없이도 충분히 실습 가능하다.

| 방법 | 설명 |
|------|------|
| 개발자 모드 로컬 로드 | `chrome://extensions/` → 압축해제된 확장 프로그램 로드 |
| `.crx` 패키징 | Chrome이 서명한 패키지, 로컬 배포 가능 (정책상 제한 있음) |
| Web Store (비공개) | `공개 범위: 비공개` 또는 `테스터 그룹`으로 제한 배포 가능 |

이 프로젝트는 우선 로컬 개발자 모드로 충분하다.
완성 후 Web Store에 올려보는 경험을 하고 싶다면 $5 등록 후 진행.
